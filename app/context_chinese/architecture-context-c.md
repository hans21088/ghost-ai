## 技術棧

| 層級             | 技術                        | 角色                                         |
| ----------------| --------------------------- | ------------------------------------------- |
| 框架             | Next.js 16 + TypeScript     | 全棧應用，具有伺服器/用戶端邊界                  |
| 使用者介         | Tailwind + shadcn/ui        | 元件組合與樣式設計                              |
| 身份驗           | Clerk                       | 使用者身份與路由保護                            |
| 資料庫           | Prisma + PostgreSQL         | 關聯式中繼資料：專案、協作者、規格、任務運行        |
| 畫布            | Liveblocks + React Flow     | 即時協作畫布、身在感與游標                       |
| 背景任務.        | Trigger.dev                 | 耐久性 AI 生成工作                             |
| 工件儲存         | Vercel Blob                 | 畫布快照與生成的 Markdown 規格                  |

## 系統邊界

- `app/api` – 經過身份驗證的請求處理程序：輸入驗證、所有權檢查、任務觸發與持久化。 
- `trigger` – 長時間運行的背景工作：AI 設計生成與規格生成。 
- `lib` – 共用基礎設施：Prisma 用戶端、存取控制協助程式與工具函式。 
- `components` – UI 組合：畫布介面、側欄、對話框與互動元素。 
- `prisma` – 資料庫架構與產生的用戶端輸出。 
- `data` – 舊版本地目錄。不用於新工件。

## 儲存模型

- **資料庫**：中繼資料、所有權、關聯性與任務運行記錄。
- **Vercel Blob**：生成的工件 – 畫布快照位於 `canvas/{projectId}.json`，規格位於 `specs/{projectId}/{specId}.md`。
- 專案記錄、規格記錄與任務運行記錄應儲存在 PostgreSQL 中。 
- 畫布內容與 Markdown 輸出儲存在 Vercel Blob 中並從中擷取。 
- Blob URL 儲存在資料庫中（`canvasJsonPath`、`filePath`）作為工件的參考。

## 身份驗證與存取模型

- 每個專案都有單一所有者（Clerk 使用者 ID）。 
- 專案可包含其他協作者。 
- 只有經過身份驗證的使用者才能存取受保護的路由。 
- 只有所有者或協作者才能變更專案資源。 
- Liveblocks 房間權杖只有在驗證專案成員資格後才會簽發。


## 入門系統設計

- 預先建立的範本是儲存在程式碼庫中的靜態畫布快照。 
- 當使用者匯入範本時，範本會載入到作用中的 Liveblocks 房間中。 
- 匯入可以在畫布建立時或從編輯器內的任何時間發生。 
- 範本資料遵循與使用者建立的畫布內容相同的節點/邊架構。 
- 範本不需要單獨的資料庫記錄；它們在匯入時以範本 ID 解析。 


## AI 生成模型

### 設計生成

- 輸入：使用者提示、專案環境與目前的畫布狀態。 
- 執行：透過 Trigger.dev 的耐久背景工作。 
- 輸出：結構化節點與邊更新寫入共用的 Liveblocks 房間。 

### 規格生成

- 輸入：目前的畫布圖表與專案環境。 
- 執行：透過 Trigger.dev 的耐久背景工作。 
- 輸出：Markdown 技術規格儲存到檔案系統並連結到資料庫中的專案。

## 不變量

1. 請求處理程序不執行長時間運行的 AI 工作 – 該工作屬於背景工作。 
2. 中繼資料與大型生成的工件儲存在不同的層級中。 
3. 身份驗證與所有權在每個變更邊界強制執行。 
4. 用戶端元件僅在瀏覽器互動或即時狀態需要時使用。 
5. 畫布架構在使用者建立的內容與匯入的範本之間必須保持一致。
